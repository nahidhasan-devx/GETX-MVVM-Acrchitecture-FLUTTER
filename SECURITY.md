## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it to us responsibly.

### How to Report

**Do not** open a public issue for security vulnerabilities. Instead, please send an email to:

ðŸ“§ **Nahid Hasan**: [security@github.com/nahidhasan-devx](mailto:security@github.com/nahidhasan-devx)

### What to Include

Please include the following information in your report:

- **Type of vulnerability**: (e.g., XSS, SQL injection, etc.)
- **Affected versions**: Which versions are affected
- **Description**: A detailed description of the vulnerability
- **Steps to reproduce**: Steps to reproduce the vulnerability
- **Impact**: The potential impact of the vulnerability
- **Proof of concept**: If applicable, a proof of concept or exploit code

### What to Expect

- We will acknowledge receipt of your report within 48 hours
- We will provide a detailed response within 7 days regarding the next steps
- We will work with you to understand and validate the report
- We will coordinate a fix and disclosure timeline with you
- We will credit you in the security advisory and release notes

### Disclosure Timeline

Once a vulnerability is confirmed:

1. We will create a security advisory in GitHub
2. We will develop a fix
3. We will coordinate the release of the fix with you
4. We will publish the security advisory after the fix is released

## Best Practices

To help keep your application secure, please follow these best practices:

- Keep dependencies updated
- Use secure storage for sensitive data
- Validate and sanitize all user inputs
- Use HTTPS for all network communications
- Implement proper authentication and authorization
- Keep your Flutter SDK updated
- Follow OWASP security guidelines
- Review code regularly for security issues
